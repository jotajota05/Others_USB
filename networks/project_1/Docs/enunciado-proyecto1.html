Primer Proyecto (10%)

El objetivo del proyecto es implementar un sistema cliente-servidor con un
protocolo que permita colocarle a la computadora donde se ejecuta el cliente
la hora que tiene la computadora donde se ejecuta el servidor.

Se debe implementar empleando los protocolos UDP y TCP.

La operación básica consiste en que el cliente debe preguntar varias
veces la hora al servidor, y hallar un "promedio" utilizando el
algoritmo de Marzullo (explicado en
http://en.wikipedia.org/wiki/Marzullo's_algorithm).

La sintaxis de la aplicación cliente es:

c_tiempo -s <servidor> -p <puerto> -v <numero de preguntas> {-t|-u} [-n]

donde:

 -s <servidor> Indica la dirección IP o nombre de host del servidor con el
               cual se desea sincronizar, donde debe estar ejecutándose la
               aplicación servidor.

 -p <puerto> Indica el puerto a través del cual se debe contactar a la aplicación
             servidor.

 -v <numero de preguntas> Indica el numero de veces que el cliente preguntará
                          la hora al servidor.

 -t Indica que se desea utilizar el protocolo TCP. La aplicación servidor debe
    estar ejecutándose empleando el mismo protocolo.

 -u Indica que se desea utilizar el protocolo UDP. La aplicación servidor debe
    estar ejecutándose empleando el mismo protocolo.

 -n Indica que la hora obtenida debe mostrarse por la salida estándar, en lugar
    de tratar de establecerla como hora del sistema (útil si no se tienen privilegios
    de administración).

La sintaxis de la aplicación servidor es:

s_tiempo -p <puerto> {-t|-u}

 -p <puerto> Indica el puerto a través del cual se aceptarán peticiones de
             la aplicación cliente.

 -t Indica que se desea utilizar el protocolo TCP. Se contestarán peticiones de
    la aplicación cliente utilizando TCP.

 -u Indica que se desea utilizar el protocolo UDP. Se contestarán peticiones de
    la aplicación cliente utilizando TCP.

Observe que todos los parámetros son obligatorios excepto "-n", y las
opciones "-t" y "-u" son mutuamente excluyentes. Se recomienda
utilizar los últimos 5 dígitos del número de carnet de uno de los
integrantes de cada equipo como número de puerto, para evitar
conflictos con otros equipos.

Si se emplea el protocolo UDP, la aplicación cliente debe preguntar
por la hora a la aplicación servidor el número de veces especificado
en la linea de comandos, utilizando un (1) paquete UDP en cada
oportunidad. En el caso en que para una pregunta no se obtenga
respuesta (no llega el paquete de respuesta), el cliente debe ignorar
esa pregunta y continuar con la siguiente.

Si se emplea el protocolo TCP, la aplicación cliente debe abrir una
conexión, enviar un mensaje y esperar una respuesta por la misma
conexión. Se debe mantener una única conexión independientemente del
numero de preguntas especificadas en la linea de comandos.

Cualquiera que sea el método seleccionado por el cliente, al finalizar
la ejecución del programa cliente el computador local debe mostrar la
hora obtenida y establecerla como hora del sistema donde se ejecuta,
exceptuando el caso en que haya sido invocado con la opción "-n".
Adicionalmente, debe dar algunas estadísticas como salida, en
particular, tamaño promedio del intervalo entre la realización de la
pregunta y la obtención de la respuesta y, en el caso de UDP, indicar
cuantas preguntas hubo cuyas respuestas no se recibieron.

El nivel de granularidad de la sincronización debe ser al menos de
segundos.

Para esta primera entrega, no se requiere que el servidor pueda
atender simultáneamente varias solicitudes.

La implementación debe realizarse usando la librería estándar de
sockets en C, y debe compilar y ejecutarse sin problemas en cualquier
estación de trabajo del LDC. La implementación debe manejar
adecuadamente situaciones excepcionales (errores) en las llamadas al
sistema. Deben seguirse los principios de modularidad, estilo y
documentación (comentarios) en el código fuente implementado.

El proyecto deberá entregarse por aula virtual, subiendo un archivo
tar.gz cuyo nombre sea p1-g<GRUPO>.tar.gz en el area de documentos de
grupo.  Dentro del archivo debe existir un directorio llamado de la
misma forma con los fuentes y un archivo Makefile que permita compilar
el proyecto (tanto cliente como servidor) ejecutando simplemente el
comando make (por ejemplo, si es el grupo 10, el archivo deberá
llamarse p1-g10.tar.gz y contener un único directorio llamado p1-g10
dentro del cual se encuentre el código fuente y el archivo
makefile). Los proyectos que no cumplan con estas reglas tendrán
penalización en la nota final. Adicionalmente se debe entregar un
archivo PDF de 3 páginas máximo que contenga un manual de uso, el
diseño básico de la herramienta y dos diagramas de secuencia que
muestren la interacción entre cliente y servidor en ambos protocolos
(TCP y UDP).
